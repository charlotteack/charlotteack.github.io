使用fragment实现页面切换

发布于 2020-03-15

github地址

简单的tabbar页面切换

实现的效果如图（这里只展示两个页面，其他的一样）


实现

首先是基本的UI设计

  • 封装top标题栏
    在主活动里面使用requestWindowFeature(Window.FEATURE_NO_TITLE);
    该方法可以取消系统默认的标题，需要引入import android.view.Window;
    接着创建一个top.xml文件，并在里面添加一个textview修改其样式即可，没什么多说的。

相应代码：

<?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="49dp"
    android:background="#000"
    android:orientation="horizontal">

    <TextView
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        android:gravity="center"
        android:text="@string/title"
        android:textColor="#fff"
        android:textSize="24sp" />

</LinearLayout>

注意这里有一个坑，由于是第一次使用requestWindowFeature方法，没有在意书写位置。
编译不会飘红，但运行会报错。解决方法是把requestWindowFeature放在setContentView方法前。

  • 封装底部tabbar
    这里很重要的一点就是可以将布局文件想象成div一样使用。
    大致思路是创建一个tabbar.xml文件，作为整个大的div，里面包含四个功能按键，所以整体的大布局显然应该是水平的。接着在里面创建相应的小div，每个小div都是由图片和文字组合而成，所以小div布局应该是垂直的。

相应代码（只放了消息部分）：

<?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="65dp"
    android:background="@drawable/bottom_bar"
    android:orientation="horizontal"
    android:baselineAligned="false">
  <!--消息-->
    <LinearLayout
        android:id="@+id/msg"
        android:layout_width="0dp"
        android:layout_height="match_parent"
        android:orientation="vertical"
        android:layout_weight="1">

        <ImageButton
            android:id="@+id/msg_imageButton"
            android:layout_width="match_parent"
            android:layout_height="wrap_content"
            android:clickable="false"
            android:background="#000"
            android:contentDescription="@string/message"
            android:src="@drawable/tab_weixin_normal" />

        <TextView
            android:id="@+id/msg_text"
            android:layout_width="match_parent"
            android:layout_height="match_parent"
            android:gravity="center"
            android:text="@string/message"
            android:textColor="#fff"
            android:textSize="16sp" />
    </LinearLayout>

    </LinearLayout>

注意ImageButton的clickable需要设置成false；图片地址系统默认生成的是app:srcCompat，我改成了android:src。

  • 页面中间的帧布局
    这个没啥好说的，就是在主页面的布局文件上面拖一个帧布局即可。
    最后主活动布局文件代码如下：
      <?xml version="1.0" encoding="utf-8"?>
  <LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
      android:layout_width="match_parent"
      android:layout_height="match_parent"
      android:orientation="vertical">

  <include layout="@layout/top"/>

  <FrameLayout
      android:id="@+id/content"
      android:layout_width="match_parent"
      android:layout_height="0dp"
      android:layout_weight="1">
  </FrameLayout>

  <include layout="@layout/bottom"/>
  </LinearLayout>

注意 这里的帧布局必须设置layout_weight，不然会盖住下面的tabbar。

然后实现不同页面之间的跳转

这是这次的难点也是重点。大概分为如下步骤：大致的思路是，把每一个xml文件通过fragment的包装变成一个对象，这样可以在主活动里面直接使用。

  • 创建每个页面对应的xml文件。在里面放一个简单的textView即可，主要是测试时能分辨不同的页面用，不多赘述。

  • 创建每个页面对应fragment文件。由于在主活动java文件里面无法直接使用其他的xml文件，需要使用fragment将其“包装”一下，以对象的方式引入。

fragment文件代码如下：

public class MessageFragment extends Fragment {

  public MessageFragment() {
      // Required empty public constructor
  }
  @Override
  public View onCreateView(LayoutInflater inflater, ViewGroup container,
                           Bundle savedInstanceState) {
      return inflater.inflate(R.layout.message, container, false);
  }

}

  • 将碎片和需要用到的控件以变量的方式引入。碎片需要调用构造函数，控件根据id即可。

  • 将需要切换的碎片添加帧布局

    代码如下：（这里content就是帧布局的id，另外需要创建一个管理碎片的FragmentManager，它可以通过transaction操作碎片的切换，最后记得需要提交）

    //添加可切换的碎片到帧布局上面
  private void initFragment() {
      fm = getFragmentManager();
      FragmentTransaction transaction = fm.beginTransaction();
      transaction.add(R.id.content, msg_fragment);
      transaction.add(R.id.content, frd_fragment);
      transaction.add(R.id.content, addr_fragment);
      transaction.add(R.id.content, set_fragment);
      transaction.commit();
  }

  • 点击切换碎片。大致思路是，先给tabbar里面每个小部分人为地设置一个编号，如0，1，2，3。当点击上面的元素时，在点击事件里面通过view.getId()判断点击的是哪个元素，再调用修改样式的方法，改变相应点击元素的图片和文字。

代码如下：

//切换碎片
    private void selectFragment(int i) {
        FragmentTransaction transaction = fm.beginTransaction();
        hideFragment(transaction);
        switch (i) {
            case 0:
                transaction.show(msg_fragment);
                msg_image.setImageResource(R.drawable.tab_weixin_pressed);
                msg_txt.setTextColor(Color.parseColor("#00ff00"));
                break;
            case 1:
                transaction.show(frd_fragment);
                frd_image.setImageResource(R.drawable.tab_find_frd_pressed);
                frd_txt.setTextColor(Color.parseColor("#00FF00"));
                break;
            case 2:
                transaction.show(addr_fragment);
                addr_image.setImageResource(R.drawable.tab_address_pressed);
                addr_txt.setTextColor(Color.parseColor("#00FF00"));
                break;
            case 3:
                transaction.show(set_fragment);
                set_image.setImageResource(R.drawable.tab_settings_pressed);
                set_txt.setTextColor(Color.parseColor("#00FF00"));
                break;
            default:
                break;
        }
        transaction.commit();
    }

注意在切换的时候先隐藏所有的碎片，再通过判断显示点击的碎片内容。还有，这里使用Color.parseColor设置颜色的时候，必须是6位，有点奇怪，也算是个坑吧。另外关于点击事件可以直接使主活动implements View.OnClickListener，但这样会监听整个页面，所以还需要指定监听的对象，即tabbar。

剩下的代码：

//点击事件
    @Override
    public void onClick(View view) {
        resetImage();
        switch (view.getId()) {
            case R.id.msg:
                selectFragment(0);
                break;
            case R.id.frd:
                selectFragment(1);
                break;
            case R.id.addr:
                selectFragment(2);
                break;
            case R.id.set:
                selectFragment(3);
                break;
            default:
                break;
        }
    }

    //只启动tabbar的事件监听
    private void initEvent() {
        msg_layout.setOnClickListener(this);
        frd_layout.setOnClickListener(this);
        addr_layout.setOnClickListener(this);
        set_layout.setOnClickListener(this);
     }

注意点击时还需要将所有的图片和文字恢复正常。
